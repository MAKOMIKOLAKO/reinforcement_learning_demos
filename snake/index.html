Snake · Deep Q-Network

A DQN agent estimates a Q-value for each of the four moves and greedily plays whichever move scores highest. Scrub the training progress slider to see the same network at different points in training, from random flailing to confident food-seeking.

Environment

Before looking at the network, it helps to pin down exactly what the agent sees, what it can do, and what it's rewarded for. The descriptions below are precise enough to reimplement the state encoding from scratch.

State space

The network receives an 11-dimensional vector, [danger_straight, danger_right, danger_left, moving_left, moving_right, moving_up, moving_down, food_left, food_right, food_up, food_down], recomputed fresh every step:

    Action space

    4 discrete actions, one per absolute compass direction:

    • up — move one cell up
    • down — move one cell down
    • left — move one cell left
    • right — move one cell right

    These are absolute directions, not relative turns — picking the action opposite the snake's current heading is a legal (if usually fatal) move, same as any other.

    Reward function

    • +10 for eating food
    • -10 for dying (hitting a wall or its own body)
    • -0.01 per step otherwise

    The small per-step penalty discourages stalling — without it, an agent that's unsure what to do could circle forever at zero cost.

    How it works

    DQN learns a single value network that outputs a Q-value for every possible action given the current state — there's no separate policy network. Acting is just argmax: at every step the agent picks whichever action the network scores highest, with no sampling involved.

    The 11-value input encodes danger one cell ahead/left/right of the snake's heading, the current heading itself, and which absolute direction the food is in.
